Django Tweet App

Full-Stack Social Platform

The Problem

To truly understand full-stack web development, I needed to build something requiring real complexity: user authentication, data relationships, permission controls, and polished UI. Tutorials weren't enough.

My Approach

I designed a complete social platform using Django's ORM. Key design decisions:

  • Permission Model: Users can only edit/delete their own posts. This rule taught me about data integrity and security.
  • Responsive Design: Built with Bootstrap for all devices, treating UI as a first-class concern.
  • Complete CRUD: Every operation implemented to understand full user lifecycle.

What I Learned

Great software requires thinking about users. Permission controls exist because users need safety. Responsive design matters because people use products on phones. These insights shaped how I approach every project.

Tech Stack: Django, Python, SQLite, Bootstrap

AI Active Routes

Unified AI Provider Platform

The Problem I Identified

AI tools proliferate—Groq, Cohere, Cerebras, Mistral—but users face friction: to find the best tool, they must sign up, test, and switch between platforms. What if there was a single place to experiment with all?

My Solution

I built a production platform that routes content generation intelligently across 4 AI providers:

  • Unified Interface: One request; system selects best provider or fails over automatically.
  • Reliability First: Advanced load balancing for 99.8% uptime with sub-2s response times.
  • User Choice: System lets users understand which provider works best for their needs.

Design Decisions

I chose those 4 providers because they each have distinct strengths: Groq for speed, Cohere for consistency, Cerebras for enterprise reliability, Mistral for cost-efficiency. The failover system isn't just metrics—it's about user trust.

Tech Stack: Node.js, Express, Load Balancing, 4 AI APIs

Performance: 99.8% uptime, sub-2s response times
